# Emberforge Dist

Public distribution bucket for **Emberforge**. This repo holds **GitHub Releases
only** — it does not hold source, and (aside from this README and `.gitignore`)
its tree stays intentionally empty. All player-facing artifacts arrive as
**release assets**, published automatically by the private source repo's CI.

## What lands here (as Release assets, not committed files)

Published by `release-launcher.yml` in the private `emberforge` source repo:

- `Emberforge_Client.jar` — the built client
- `emberforge-launcher-manifest.json` — client update manifest
  (`product / version / generatedAt / files[]` with SHA-256 + size)
- `VERSION`, `android_version.txt`
- The Windows installer (`.exe`) and `latest.yml` — launcher self-update feed
  (electron-updater)

## Who reads them

- **The launcher** fetches
  `https://github.com/Cooperman00/emberforge-dist/releases/latest/download/emberforge-launcher-manifest.json`
  to decide whether to download/verify/swap the client.
- **electron-updater** reads `latest.yml` from the same `releases/latest` feed to
  self-update the launcher.

## Source of truth (all in the private `emberforge` repo — do not duplicate here)

- Manifest generator: `scripts/generate-release-manifest.js`
- Publish pipeline: `.github/workflows/release-launcher.yml`
- Launcher/updater client: `launcher/main.js`
- Design + ticket: `docs/superpowers/specs/2026-06-29-launcher-distribution-design.md`,
  `workflow/tickets/TICKET-046-launcher-distribution.md`

## To publish

Bump `/VERSION` on `main` in the source repo (or run the workflow manually). CI
builds and uploads the release here. This requires the repo secret
`EMBERFORGE_DIST_TOKEN` (a PAT with `repo` scope that can write Releases to this
repo) to be configured in the source repo.

> Do not hand-commit a `manifest.json` into this repo — the authoritative
> manifest is `emberforge-launcher-manifest.json`, generated by the source repo
> and served from Releases. A committed file in the tree is not read by anything.
